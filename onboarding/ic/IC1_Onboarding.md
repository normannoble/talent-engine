# 30-60-90 Day Onboarding: Associate Software Engineer (IC1)

## Overview

**Level:** IC1 — Associate Software Engineer
**Focus:** Learning fundamentals, becoming productive, integrating into team
**Manager's Role:** High-touch support, clear guidance, structured learning

IC1s are early in their career and need more structure and support. The goal is to build foundational skills while contributing meaningfully to the team.

---

## Days 1-30: Learn

### Theme: Build Foundation

**Primary Objectives:**
- Understand the team, codebase, and development workflow
- Build relationships with teammates
- Complete initial small tasks with guidance
- Learn company and team norms

### Week 1: Orientation

| Day | Focus | Activities |
|-----|-------|------------|
| 1 | Welcome | Team introductions, workspace setup, HR orientation |
| 2 | Context | Team mission, current projects, how your team fits in org |
| 3 | Technical | Development environment setup, codebase walkthrough |
| 4 | Process | Git workflow, PR process, deployment pipeline overview |
| 5 | Connect | 1:1s with 2-3 teammates, onboarding buddy check-in |

**Key Meetings to Schedule:**
- [ ] Daily check-ins with manager (15 min)
- [ ] Onboarding buddy sync (30 min, 2x/week)
- [ ] Team standup introduction
- [ ] 1:1 with each teammate

### Weeks 2-4: Immersion

**Learning Goals:**
- [ ] Understand the architecture at a high level
- [ ] Know where to find documentation and answers
- [ ] Be able to run the application locally
- [ ] Complete first small tasks (bug fixes, small features)
- [ ] Understand team ceremonies (standup, retro, planning)

**Activities:**
- Shadow teammates on their work
- Pair program with senior engineers
- Complete onboarding tasks/tutorials
- Start contributing to code reviews (as a learning exercise)
- Ask lots of questions—this is expected and encouraged

**Technical Milestones:**
- [ ] Development environment fully working
- [ ] First PR submitted (even if small)
- [ ] First PR merged
- [ ] Can navigate the codebase to find relevant code
- [ ] Understands testing approach

### 30-Day Check-in

**Manager Discussion Points:**
- How is learning going? What's clear vs. confusing?
- Are you getting enough support?
- How are relationships with teammates?
- What questions do you have?
- Feedback on first contributions

**Success Indicators:**
- [ ] Comfortable asking questions
- [ ] First contributions merged
- [ ] Knows teammates and their areas
- [ ] Can complete small tasks with guidance
- [ ] Understands team workflow

---

## Days 31-60: Contribute

### Theme: Build Independence

**Primary Objectives:**
- Take on increasingly complex tasks
- Require less hand-holding on familiar work
- Start building technical depth in one area
- Contribute meaningfully to code reviews

### Weeks 5-6: Initial Independence

**Activities:**
- Own small features end-to-end (with support available)
- Write tests for your code
- Participate actively in code reviews
- Start documenting what you learn
- Ask clarifying questions before starting work

**Technical Growth:**
- [ ] Completing tasks with less guidance
- [ ] Asking better questions (more specific, after attempting)
- [ ] Understanding why, not just how
- [ ] Catching basic issues in code reviews

### Weeks 7-8: Building Momentum

**Activities:**
- Take on slightly larger scope
- Debug issues more independently
- Contribute ideas in team discussions
- Help onboard any newer team members
- Start identifying areas of interest

**Expanding Skills:**
- [ ] Comfortable with standard task types
- [ ] Can estimate simple tasks reasonably
- [ ] Knows when to ask for help vs. push through
- [ ] Starting to develop opinions on code quality

### 60-Day Check-in

**Manager Discussion Points:**
- What are you most proud of so far?
- Where do you still feel uncertain?
- How is your pace of learning?
- What kind of work do you enjoy most?
- Feedback on independence and quality

**Success Indicators:**
- [ ] Delivering small features with minimal guidance
- [ ] Code quality is improving
- [ ] Asking for help appropriately (not too much, not too little)
- [ ] Integrated into team workflow
- [ ] Beginning to show areas of strength

---

## Days 61-90: Own

### Theme: Establish as Contributor

**Primary Objectives:**
- Operate as a productive team member
- Handle standard tasks independently
- Show consistent quality in work
- Demonstrate learning orientation

### Weeks 9-10: Full Contribution

**Activities:**
- Own tasks from start to finish
- Participate fully in planning discussions
- Provide useful code review feedback
- Help teammates when you can
- Take initiative on small improvements

**Demonstrating IC1 Expectations:**
- [ ] Reliable delivery on assigned work
- [ ] Quality code that passes review efficiently
- [ ] Clear communication about progress and blockers
- [ ] Active participation in team activities
- [ ] Continuous learning and improvement

### Weeks 11-12: Establishing Trajectory

**Activities:**
- Look ahead to next quarter's work
- Identify areas for continued growth
- Set development goals with manager
- Build on strongest areas
- Transition from "new hire" to "team member"

**Preparing for Ongoing Success:**
- [ ] Development plan drafted
- [ ] Clear on IC1 expectations
- [ ] Understand path to IC2
- [ ] Relationships solidified
- [ ] Comfortable in role

### 90-Day Review

**Manager Discussion Points:**
- Overall assessment of onboarding success
- Strengths demonstrated
- Areas for continued development
- Goals for next quarter
- Feedback and recognition

**Success Indicators:**
- [ ] Completing tasks independently at IC1 level
- [ ] Integrated and contributing team member
- [ ] Quality of work meets standards
- [ ] Learning trajectory is positive
- [ ] Ready for ongoing performance management

## Key Relationships to Build

| Who | Why | When to Meet |
|-----|-----|--------------|
| Direct manager | Primary support and feedback | Daily → Weekly |
| Onboarding buddy | Peer support, questions, culture | Multiple times per week |
| Team members | Collaboration, learning | First 2 weeks, ongoing |
| Tech lead/Senior | Technical guidance | As needed |

---

## Common IC1 Onboarding Challenges

| Challenge | Solution |
|-----------|----------|
| Afraid to ask questions | Normalize questions; create safe space; praise good questions |
| Overwhelmed by codebase | Focus on one area first; provide guided tours; pair programming |
| Imposter syndrome | Set appropriate expectations; celebrate small wins; provide reassurance |
| Unclear on expectations | Document clear expectations; frequent check-ins; explicit feedback |
| Struggling with basics | More pairing; targeted learning resources; patience |

---

## Manager Checklist

### Pre-Arrival
- [ ] Equipment and accounts ready
- [ ] Onboarding buddy assigned
- [ ] First week calendar prepared
- [ ] Initial tasks identified (small, achievable)
- [ ] Team notified of new hire

### Week 1
- [ ] Welcome and introductions complete
- [ ] Development environment working
- [ ] Daily check-ins scheduled
- [ ] Codebase walkthrough done
- [ ] First task assigned

### Ongoing
- [ ] Regular feedback provided
- [ ] Learning pace appropriate
- [ ] Support level right (not too much, not too little)
- [ ] Early wins celebrated
- [ ] Challenges addressed promptly
